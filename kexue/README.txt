红黑树第二版比起第一版区别：
在rb_tree_node_base中增加了node_count表示：以当前节点为根的子树的大小。
作用：通过增加这个属性，可以实现红黑树logn时间实现：1.根据值获取它的排名(排名的定义：对于x，小于等于x的个数+1 ) 2.根据排名获取某个值
然后可以通过get_rank函数得到类型为unsigned long long 的排名(由定义可知，这个值必然大于等于1)；通过get_val函数得到排名对应的iterator,如果排名大于size或者排名等于0，那么返回end()，如果小于0，因为强转型的原因不一定返回什么。

封装成了两个：
1.map 不允许重复插入，允许[]运算符
2.multimap 允许重复插入，不允许[]运算符,对count函数没有仿造stl，而是进行了重写，保证了logn的时间复杂度
